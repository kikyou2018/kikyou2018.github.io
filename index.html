1. HTTPS

工作原理

  1. 客户端发起HTTPS连接：客户端通过URL向服务器发送HTTPS请求。通常是通过443端口。这个请求是明文的，但是服务器会提供一个加密通信的方案。
  2. 服务器发送证书：服务器收到请求后，会返回一个SSL/TLS证书（包括服务器的公钥、证书的有效期、颁发机构等信息），用于加密数据。
  3. 客户端验证证书：客户端验证证书。验证通过，客户端会随机生成一个随机的对称密钥，并用服务器提供的公钥加密这个对称密钥。
  4. 服务器解密密钥：服务器使用自己的私钥解密，得到客户端生成的对称密钥。
  5. 加密通信：双方获得对称密钥后，之后的通信都通过对称加密算法进行加密传输，确保数据在传输过程中不会被窃听和篡改。

TCP和UDP的头部格式

  1. UDP四个字节。分别是源端口号、目的端口号、包长度（头部+数据）、校验和。分别占2个字节。共占8个字节
  2. TCP20个字节：源端口号、目的端口号、序列号、确认号、首部长度、保留位（6位）、标志字段（6位）、窗口大小、校验和、紧急指针。共占20个字节（最少）

IP地址分类

IP地址分为5类，其中

  • A类0.0.0.0-127.255.255.255
  • B类128.0.0.0- 191.255.255.255
  • C类192.0.0.0-223.255.255.255
    其中，主机号全为1指定在某个网络下的所有主机，用于广播。主机号全为0则表示指定某个网络。
  • D类地址用于多播。E类未使用。多播地址用于将包发送给特定组内的所有主机。由于广播无法穿透路由，若想给其他网段发送同样的包，就可以使用可以穿透路由的多播。

为什么要分离网络号和主机号

因为两台计算机要通讯，首先要判断是否处于同一个广播域中，即网络地址是否相同。如果网络地址相同，表明接受方在本网络上，那么可以把数据包直接发送到目标主机。路由器寻址过程中，也就是通过这样的方式来找到对应的网络号的，进而把数据包转发给对应的网络内。

什么是公有IP地址，什么是私有IP地址

A类地址的范围是0.0.0.0-127.255.255.255，其中私有IP地址范围是10.0.0.0-10.255.255.255
B类地址的范围是128.0.0.0-191.277.277.277，其中私有IP地址范围是172.16.0.0-172.31.255.255
C类地址的范围是192.0.0.0-223.255.255.255,其中私有IP地址范围是192.168.0.0-192.168.255.255

IP地址和路由转换

IP地址的网络地址这一部分是用于进行路由控制。路由控制表中记录着网络地址与下一步应该发送至路由器的地址。在主机和路由器上都会有各自的路由控制表。
在发送IP包时，首先要确认IP包首部中的目标地址，再从路由控制表中找到与该地址具有相同网络地址的记录，根据该记录将IP包转发给相应的下一个路由器。如果路由控制表中存在多条相同网络地址的记录，就选择相同位数最多的网络地址，也就是最长匹配。

IP分片和重组

每种数据链路的最大传输单元MTU是不相同的，以太网是1500。当IP数据包大于MTU时，IP数据包就会被分片。讲过分片之后的IP数据报在重组的时候，只能由目标主机进行，路由器是不会进行重组的。在分片传输中，一旦某个分片丢失，则会造成整个IP数据报作废，所以TCP引入了MSS，也就是在TCP层进行分片不由IP层分片，对于UDP尽量不要发送一个大于MTU的数据报文。

MSS（最大报文长度）的作用

优化性能：减少数据分片

MSS的确定

通常取决于数据链路层的最大传输单元MTU。MTU指的是在一个网络层上传输的数据报文的最大长度。MSS=MTU-TCP头部（20字节）-IP头部（20字节）= 1460字节

  1. 使用IPv6
  2. NAT穿透技术：NAT穿透技术能够让网络应用程序主动发现自己位于NAT设备之后，并且会主动获得NAT设备的公有IP，并为自己建立端口映射条目，注意这些都是NAT设备后的应用程序自动完成的。也就是说，客户端主动从NAT设备获取公有IP地址，然后自己建立端口映射条目，然后使用这个条目对外通信，就不需要NAT设备来进行转换了。

ICMP Internet control message protocol互联网控制报文协议

ICMP互联网控制消息协议，网络层的一种协议，主要用于在IP网络中传递控制信息和错误报告。ICMP并不是用于传输应用层数据，而是为网络设备（如路由器、主机）之间提供诊断和网络状态信息的辅助协议。在IP协议之上工作，通常用来报告网络连接中的问题。
主要功能：

  1. 错误报告：当某个数据包无法成功送达目的地时，ICMP会发送一条错误消息来告知源主机。例如，如果一个路由器发现某条路由不可达，它会通过ICMP向发送方报告这个问题。常见的错误信息包括：
    • 目的不可达：数据包无法达到目的地
    • 时间超时：数据包在网络中停留的事件超过了IP头部中的TTL（生存时间）字段限制
    • 重定向消息：建议主机改变数据包发送的路由
  2. 网络诊断
    • ping命令。最常用的ICMP工具之一，利用ICMP的“回显请求”和“回显应答”消息来测试两个节点之间的连通性。源主机发送ICMP回显请求消息，目标主机应答以表明其可达性。
    • traceroute：另一个常见的诊断工具，利用ICMP“时间超时”消息来跟踪数据包从源到目的地经过的路由。通过逐步增加数据包的TTL来获取中间节点的IP地址。
  3. ICMP消息的种类：ICMP消息分为不同类型，每个类型对应一种不同的功能。常见的ICMP消息类型包括：
    • 回显请求和回显应答（类型 8 和 0）：用于ping操作来检查网络连通性。
    • 目的不可达（类型 3）：用于表示某个数据包无法送达目的地。
    • 时间超时（类型 11）：用于traceroute，当数据包在网络中的TTL到期时生成此消息。
    • 重定向消息（类型 5）：通知主机应该将数据包发送到另一个路由器。
  4. 特点：
    • ICMP是无连接的协议，不需要维护连接状态
    • 不可靠传输：ICMP本身不保证消息的可靠送达。
  5. ICMP与IP的关系：ICMP被视为IP协议的扩展，主要用于报告网络错误行为和进行控制信息传递。每当网络层（IP层）遇到问题时，通常通过ICMP来通知源主机。

IGMP因特网组管理协议

D类地址，也就是组播地址。IGMP工作在主机（组播成员）和最后一跳路由之间。IGMP报文向路由器申请加入和退出组播组，默认情况下路由器是不会转发组播包到连接中的主机，除非主机通过IGMP加入到组播组，主机申请加入到组播组时，路由器就会记录IGMP路由表，路由器后续就会转发组播包到对应的主机了。IGMP报文采用IP封装，IP头部的协议号为2，TTL字段为1.
注 ：组播地址不是用于及其IP地址的，因为组播地址没有网络号和主机号，所以和DHCP没关系。组播地址一般是用于UDP协议，及其发送UDP组播数据时，目标地址填的是组播地址，那么在组播组内的及其都能收到数据包。是否加入组播组和离开组播组，是由一个socket接口实现的，主机ip是不用改变的。

IP包头

  • 版本号：4位
  • 首部长度：4位
  • 服务类型TOS：8位。服务类型字段用于指示IP数据包的服务质量要求。可以告诉网络中的路由器如何处理该数据包，以便为不同类型的流量提供不同的服务质量。主要由：1.优先级：用于区分不同优先级的数据包。优先级越高的数据包在网络中会得到优先处理。2.延迟、吞吐量和可靠性：分别对数据包传输延迟、吞吐量和可靠性的要求。例如对于实时性要求较高的语音或视频流量，可以设置低延迟；对于文件传输等大数据量的流量，可以设置高吞吐量。
  • 总长度：16位
  • 标识：16位。用于唯一标识一个IP数据包。当一个大的数据包被分割成多个小的片段（分片）进行传输时，这些分片都具有相同的标识值，以便接收方能够正确地重组这些分片。
  • 标志：3位。第一位为保留位。第二位：不分片。第三位：更多分片位。标识之后还有同一个数据包的分片。
  • 片偏移：13位
  • TTL：8位
  • 协议：8位。协议字段用于指示IP数据包所携带的数据部分使用的上层协议类型。其中，值为6表示TCP协议，值为17表示UDP协议。
  • 首部校验和：16位
  • 源IP地址：32位
  • 目标IP地址：32位

网卡

数据在加上MAC头部的数据包之后，将数字信息转换为电信号，才能在网线上传输。由网卡负责这一操作，要控制网卡还需要网卡驱动程序。
网卡驱动从IP模块获取到包之后，会将其复制到网卡内的缓冲区中，接着会在其开头加上报头和起始帧分界符，在末尾加上用于检测错误的帧校验序列。
最后网卡将包转换为电信号。

题目描述：
给你两个字符串 s 和 t ，统计并返回在 s 的 子序列 中 t 出现的个数，结果需要对 10^9 + 7 取模。
示例 1：
输入：s = “rabbbit”, t = “rabbit”
输出：3

示例 2：
输入：s = “babgbag”, t = “bag”
输出：5

提示：
1 <= s.length, t.length <= 1000
s 和 t 由英文字母组成

解题法：

  1. 回溯法。
    首先找到t的第一个字符匹配的第一个地方。，然后计算后面字符的匹配的count。然后计算t的第一个字符匹配的第二个地方，计算后面字符匹配的count… 但是时间复杂度很高，具体复杂度大概为(s.size()/26)^n。

正确做法是使用动态规划的做法。定义一个二维数组dp[i][j]表示s[0,i-1]与t[0,j-1]的匹配数量
递推公式
如果 s[i-1] == t[j-1]，那么 dp[i][j] = dp[i-1][j-1] + dp[i-1][j]，即当前字符匹配时，可以选择匹配它或者跳过它。
如果 s[i-1] != t[j-1]，那么 dp[i][j] = dp[i-1][j]，即跳过 s[i-1]，继续匹配。
初始值：找到第一个字符匹配的位置k，那么之前的所有dp[][]均为0，dp[k+1][k+1]为1，之后就进行动态规划即可。

注意：
对于二维动态规划，dp[i][j]均要使用两次for循环。
对于初始值的选择，要考虑特殊情况。
比如为空、边界等。
对于i j循环哪一个是外层循环，哪一个是内层循环，主要根据递推公式来进行考虑。
对于上面这个例子， 递推公式是

  1. 最长回文子序列
    给你一个字符串 s ，找出其中最长的回文子序列，并返回该序列的长度。
    子序列定义为：不改变剩余字符顺序的情况下，删除某些字符或者不删除任何字符形成的一个序列。
    思路：
  2. 状态定义：dp[i][j]表示子串s[i…j]的最长回文子序列的长度。
  3. 状态转移方程：
    • 如果s[i] == s[j]，那么 dp[i][j] = dp[i+1][j-1] + 2。
    • 如果 s[i] != s[j]，那么 dp[i][j] = max(dp[i+1][j], dp[i][j-1])。
  4. 初始化条件
    • 当i==j时，表示单个字符，dp[i][j] = 1;
  5. 最终条件答案保存在dp[0][n-1]中，其中n是字符串的长度。 注意
  • 尤其注意递推公式的划分。对于s[i] == s[j]，那么 dp[i][j] = dp[i+1][j-1] + 2，情况显而易见。对于s[i] != s[j]，那么 dp[i][j] = max(dp[i+1][j], dp[i][j-1])，要懂得进行分类。
  • 尤其注意动态规划的解法本质上就是先计算子问题的解并记录下来，然后再根据子问题的解和递推公式求解新问题。所以循环的设定应该是问题由小到大。对于上面不同的子序列。其递推公式是dp[i][j] = dp[i-1][j-1] + dp[i-1][j]和 dp[i][j] = dp[i-1][j]，为了求得dp[i][j]的值，需要先求得dp[i-1][j-1]和dp[i-1][j]的值，所以i = 1m, j = 1n的循环完全可以实现先求解子问题。
