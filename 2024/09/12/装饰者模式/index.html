装饰者模式

2024/9/12 STUDY 设计模式

重要性程度：3
动态地给一个对象增加一些额外的职责，就增加对象功能来说，装饰模式比生成子类实现更加灵活。是一种对象结构型模式。与继承关系相比，关联关系的主要优势在于不会破坏类的封装性，而且继承是一种耦合度较大的静态关系，无法在程序运行时动态扩展。

  1. 核心组件
    1. 抽象组件
    2. 具体组件
    3. 抽象饰品
    4. 具体饰品
  2. 优点：
    1. 灵活性高。装饰模式与继承关系的目的都是要扩展对象的功能，但是装饰模式可以提供比继承更多的灵活性。可以通过一种动态的方式来扩展一个对象的功能，通过配置文件可以在运行时选择不同的装饰器，从而实现不同的行为。
    2. 遵循开放封闭原则。具体构件类与具体装饰类可以独立变化，用户可以根据需要增加新的具体构件类和具体装饰类，在使用时再对其进行组合，原有代码无须改变，符合“开闭原则”
    3. 单一职责原则。
  3. 缺点：
    1. 增加复杂性
    2. 调试困难
  4. 代码
    抽象组件：咖啡
    具体组件：黑咖啡
    抽象饰品：小料
    具体饰品：牛奶 巧克力
#include <iostream>
#include <memory>
#include <string>
using namespace std;
// 抽象咖啡
class Coffee {
public:
    virtual string getDescription() const {
        return "unknown coffee";
    }
    virtual double cost() const = 0;
    virtual ~Coffee() = default;
};

// 具体组件：黑咖啡
class BlackCoffee : public Coffee {
public:
    std::string getDescription() const override {
        return "Black Coffee";
    }

    double cost() const override {
        return 2.0;
    }
};

// 抽象饰品
class CoffeeDecorator : public Coffee {
protected:
    unique_ptr<Coffee> coffee;//被装饰对象
public:
    CoffeeDecorator(unique_ptr<Coffee> c) : coffee(std::move(c)) {};
};

// 具体装饰者：奶油
class CreamDecorator : public CoffeeDecorator {
public:
    CreamDecorator(std::unique_ptr<Coffee> c) : CoffeeDecorator(std::move(c)) {}

    std::string getDescription() const override {
        return coffee->getDescription() + ", Cream";
    }

    double cost() const override {
        return coffee->cost() + 0.5;
    }
};

// 具体装饰者：巧克力
class ChocolateDecorator : public CoffeeDecorator {
public:
    ChocolateDecorator(std::unique_ptr<Coffee> c) : CoffeeDecorator(std::move(c)) {}

    std::string getDescription() const override {
        return coffee->getDescription() + ", Chocolate";
    }

    double cost() const override {
        return coffee->cost() + 0.7;
    }
};
int main() {
    // 创建一个黑咖啡
    std::unique_ptr<Coffee> myCoffee = std::make_unique<BlackCoffee>();
    std::cout << myCoffee->getDescription() << " 价格: $" << myCoffee->cost() << std::endl;

    // 在黑咖啡上加奶油
    myCoffee = std::make_unique<CreamDecorator>(std::move(myCoffee));
    std::cout << myCoffee->getDescription() << " 价格: $" << myCoffee->cost() << std::endl;

    // 在加了奶油的咖啡上再加巧克力
    myCoffee = std::make_unique<ChocolateDecorator>(std::move(myCoffee));
    std::cout << myCoffee->getDescription() << " 价格: $" << myCoffee->cost() << std::endl;

    return 0;
}

理解：
与其说是装饰，不如理解成，一个饰品克隆了一个被装饰者，但是克隆的时候，加上了饰品。但是克隆一个肉体太麻烦，就使用移动语义仅仅是抹除了思想上的本我，然后加上饰品创造了一个新我。
以此类推。
