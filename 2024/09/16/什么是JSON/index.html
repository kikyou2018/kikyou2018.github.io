什么是JSON

Posted on 2024-09-16 Edited on 2024-09-18 In STUDY

JSON 指的是JavaScript Object Notation JS对象表示法，是一种轻量级的文本数据交换格式。JSON使用JavaScript语法来描述数据对象，但是JSON独立于语言和平台。

JSON语法

  1. 数据在名称/值对中
  2. 数据由逗号 , 分隔
  3. 使用斜杆 \ 来转义字符
  4. 大括号 {} 保存对象
  5. 中括号 [] 保存数组，数组可以包含多个对象

JSON的两种结构

  1. 对象：大括号保存，是一个无序的名称/值对集合。
  2. 数组：中括号，保存的是值的有序集合。值value可以是双引号括起来的字符串、数值、true、false、null、对象或者数组，可以嵌套。

JSON.parser()

JSON通常用于与服务端交换数据，在接收数据时一般是字符串，可以使用JSON.parse()将数据转换为JavaScript对象。

JSON的用途

  1. 数据传输：JSON最常见的用途是通过HTTP协议传输数据，尤其在web应用中用于客户端和服务器之间的数据交换。
  2. 配置文件：许多软件或工具使用JSON格式存储配置数据。
  3. 数据存储：JSON也可以用来存储结构化数据，尤其是在NoSQL数据库中，文档格式基本上就是JSON。

自定义一个JSON解析器，应该由哪些部分组成

  1. 词法分析器：将输入的JSON字符串分解为一系列的标记Token，每个Token代表JSON语法的基本组成部分，如大括号{}、方括号{}、冒号：、逗号，以及关键字和数据类型。
    • 作用：扫描输入的JSON字符串，将其切割成有意义的标记。
    • 实现细节：可以遍历字符串，判断是否是数字、字符串、布尔值、null，以及JSON语法中的特殊字符（如：，{}）
  2. 语法分析器：语法分析器负责根据词法分析器生成的标记序列，解析成树状数据结构。该结构通常是一个由对象、数组、字符串、数字等元素组成的树。
    • 作用：将JSON标记序列组织成有层次地结构（例如对象包含键值对、数组包含多个值）
    • 实现细节：通常使用递归下降解析方法，根据JSON的语法规则，构造相应的数据结构（如树或嵌套对象）
    • 具体任务：
    1. 解析对象：以{}包裹的一组键值对
    2. 解析数组：以[]包裹的一组值
    3. 解析值，解析JSON值可以是对象、数组、字符串、数字、布尔值或null
    4. 处理嵌套：JSON允许对象和数组的嵌套，因此语法分析器需要能够处理嵌套结构。
  3. 数据结构：解析器需要一个合适的数据结构来存储解析后的JSON数据。这些数据结构一般支持递归（例如树状结构），以处理JSON的嵌套关系。
    • 对象：通常使用类似于std::map或std::unordered_map的结构，键为字符串，值可以是任意JSON支持的类型。
    • 数组：通常使用类似std::vector或std::list来存储JSON数组中的元素。
    • 基础类型：包括string（字符串），double或int（数字），bool，null
  4. 错误处理：解析过程中，需要进行错误检测，并提供相应的错误提示。
    • 语法错误：如果JSON格式不正确。
    • 类型错误：值类型不符合预期，比如JSON语法要求一个对象而实际上是数组。
  5. 序列化：解析器除了解析JSON字符串外，还需要支持将内部数据结构重新转换成JSON字符串，这个过程称为序列化。
    • 作用：将解析后的数据结构重新生成JSON字符串，以便数据可以保存或传输。
    • 实现细节：根据树状数据结构，遍历每一个节点（对象或数组），并按顺序生成相应的JSON字符串。
  6. 扩展功能：格式化输出；支持注释；流失解析。

和protobuf的区别

  1. 格式和可读性
    • JSON是一种人类可读的文本格式，它使用键值对来表示对象，结构化数据简单直观，可以直接在文本编辑器中查看和编辑。常用于网络通信和配置文件。
    • Protobuf是一种二进制格式，不适合直接查看或编辑。是为数据的高效序列化和反序列化设计的，特别是在网络通信和存储方面。Protobuf更适合机器读取和处理，不适合人类直接阅读。
  2. 序列化和传输效率
    • JSON：由于是文本格式，数据较为冗长。传输数据时，JSON文件通常比二进制文件格式大很多，序列化和反序列化速度较慢，特别是当数据量很大的时候。效率低。
    • Protobuf：是二进制格式，序列化后的数据相对更加紧凑，占用更少的带宽和存储空间。由于Protobuf对数据进行了压缩优化，序列化和反序列化的效率非常高，更适合在高性能应用场景中使用，比如实时通讯、数据密集型系统。
  3. 数据结构定义
    • JSON：不需要事先定义数据结构。，数据的结构是自描述的。通过键值对可以轻松理解每个数据的含义，非常灵活，但没有强类型约束，这会导致数据在传输过程中出现不一致或解析错误。
    • Protobuf：必须预先定义数据结构，使用.proto文件定义每个数据类型的字段和规则。这种方法提供了强类型约束，确保了数据的兼容性和一致性。如果数据结构发生变化，可以通过向Protobuf schema中添加新的字段来实现向后兼容。
  4. 数据类型支持
    • JSON：支持的类型较为基础，主要包括字符串、数字、布尔值、数组、对象和null。在更复杂的场景下，可能需要自己进行扩展（例如日期、时间、二进制数据等）。
    • Protobuf：支持丰富的数据类型，除了基本的整数、浮点数、布尔值、字符串等，还支持枚举、嵌套对象、字节数组（用于存储二进制数据）等高级数据类型。同时 Protobuf 的类型系统更严格，字段可以指定为必填或可选，并且支持默认值。
